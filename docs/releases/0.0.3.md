# 0.0.3 — Storage Optimization & Backlog Discipline

Released: 2026-01-27

This release focuses on storage efficiency for AAA-scale codebases and "gentle" backlog discipline enforcement.

## Highlights

- **74.5% storage savings** with binary vector format
- **Human-readable metadata files** for easier debugging
- **vLLM support** for self-hosted embeddings
- **Soft git hooks** for backlog discipline

## New Features

### Binary Vector Storage
- Implemented `struct.pack` binary format (6 KB vs 30 KB per vector)
- Auto-detection for backward compatibility
- CLI flag: `--storage-format binary|json`
- Default: binary

### Human-Readable Metadata
- `.meta.json` files generated alongside SQLite databases
- Parses embedding_space_id into readable components
- No SQL queries needed for debugging

### vLLM & OpenAI-Compatible APIs
- Added `base_url` parameter to OpenAI adapter
- Support for vLLM, Ollama, LocalAI
- Complete setup guide in `docs/vllm-embedding-setup.md`

### Orphan Commit Detection
- Soft git hooks with gentle reminders
- Smart exemptions for trivial commits
- CLI commands: `kano-backlog orphan check|suggest`
- Easy disable: `git config kano.backlog.reminders false`

## Performance

- Storage: 836 MB → 213 MB (74.5% savings)
- Query speed: ~16ms (no regression)
- AAA-scale projection: 3M chunks = 18 GB (vs 95 GB with JSON)
